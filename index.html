Algorithms Exploration - Cubing Workflow

Module 9
This module was for sure an interesting one. At first I saw no point in doing it, I was like "What does this have to do with computer science?" However as I proceeded to do the module I began to see some links between the two things. One of the major things I took out of it was how efficiency was key to improve your time. You can't stick to one method if you want to do well, but you need to be able to analyze the cube's state and find the best algorithm to solve the cube the fastest. This carries over to coding as well. When making a program it's best to make simple code that is clean and not repetitive to make it run faster and more efficiently. This can be done by making loops and functions to help minimize repetition in code. By doing this not only does the code run faster, but it takes less space, is easier to understand, and it's easier to change or update. This module has helped me understand this concept and I plan to use it moving forward to help make my code cleaner. Here's a link to the github repo for this module.
